# 지원사업 통합 대시보드

정부지원사업 / R&D / 투자프로그램 공고를 매일 자동 수집해서 한 화면에서 보는 대시보드.

## 사용법 (로컬)

| 하고 싶은 것 | 방법 |
|---|---|
| **사이트 추가하며 쓰기 (권장)** | `앱_시작.bat` 더블클릭 → 브라우저가 자동으로 열림 |
| 그냥 공고만 보기 | `index.html` 더블클릭 |
| 지금 바로 새 공고 수집 | `업데이트.bat` 더블클릭 |
| 2시간마다 자동 수집 켜기 | `자동실행등록.bat` 더블클릭 (1회만) |

### ＋ 사이트 추가 (버튼으로)
`앱_시작.bat` 으로 열면 우측 상단에 **＋ 사이트 추가** 버튼이 있습니다.
공고 목록 페이지의 **제목과 링크만 입력**하면 그 사이트에서 공고를 자동으로 긁어옵니다.
- **🔍 미리 확인**: 실제로 몇 건이 잡히는지 미리 보여줍니다.
- **추가하기**: 소스로 등록하고 즉시 수집합니다. (이후 2시간마다 자동 수집에도 포함)
- 추가한 사이트는 같은 팝업에서 **삭제**할 수 있습니다.
- 자동수집이 0건이면 그 사이트는 구조가 특수한 경우이니, '자동 수집' 체크를 끄고 바로가기로만 추가하세요.

> `index.html`을 그냥 더블클릭하거나 GitHub Pages(폰)에서는 이 버튼이 동작하지 않습니다(크롤링을 실행할 서버가 없기 때문). 사이트 추가는 반드시 `앱_시작.bat` 으로 여세요.

## 카카오톡 공유 설정하기 (PC에서 카카오톡으로 바로 보내기)

텔레그램·링크복사는 설정 없이 바로 됩니다. **PC에서 카카오톡으로 바로 전송**하려면 무료 카카오 키가 필요합니다. (약 5분)

1. **카카오 개발자 사이트 접속** → https://developers.kakao.com → 오른쪽 위 **로그인**(카카오 계정).
2. 상단 메뉴 **[내 애플리케이션]** 클릭 → **[애플리케이션 추가하기]**.
   - 앱 이름: 아무거나 (예: `지원사업 대시보드`)
   - 사업자명/회사명: 아무거나 (예: 본인 이름) → **저장**.
3. 방금 만든 앱을 클릭 → 왼쪽 메뉴 **[앱 키]** → **JavaScript 키** 값을 복사. (영문+숫자 32자 정도)
4. 왼쪽 메뉴 **[플랫폼]** → **Web 플랫폼 등록** → **사이트 도메인**에 아래를 각각 추가하고 저장:
   - `http://localhost:8770`  ← PC 앱용 (필수)
   - `https://<본인깃허브아이디>.github.io`  ← 폰(GitHub Pages)에서도 쓰려면 추가
     (예: `https://connectvillage-hub.github.io`)
5. 왼쪽 메뉴 **[카카오 로그인]** 은 설정할 필요 없습니다. **[제품 설정] → [카카오톡 공유]** 가 있으면 **활성화 ON** 으로 둡니다. (기본 켜져 있음)
6. 프로젝트 폴더의 **`설정.js`** 파일을 메모장으로 열어, 맨 아래 줄의 따옴표 안에 3번에서 복사한 키를 붙여넣고 저장:
   ```js
   window.KAKAO_JS_KEY = "여기에_복사한_JavaScript_키";
   ```
7. `앱_시작.bat` 을 다시 실행하거나, 브라우저에서 **Ctrl+Shift+R**. 이제 **공유 → 카카오톡** 버튼이 카카오톡으로 바로 전송됩니다.

> JavaScript 키는 원래 브라우저에 공개되는 값이라(도메인 등록으로 보호됨) `설정.js` 에 넣어도 안전합니다.
> "앱 도메인 등록이 필요합니다" 오류가 뜨면 4번의 도메인이 지금 접속한 주소와 정확히 같은지 확인하세요(`http://localhost:8770`).

- 공고 제목을 클릭하면 **사이트 안에서 안내 팝업**이 열립니다 — 원본 공고 내용을 긁어 보여주고,
  **공고문·첨부파일을 바로 내려받고**, 원본 링크로도 갈 수 있습니다. (앱_시작.bat 실행 시)
  그냥 index.html 로 열었을 때는 클릭하면 원본이 새 탭으로 열립니다.
- 공고를 클릭하면 **읽음(회색)** 처리됩니다.
- 카드의 **☆** 를 누르면 저장(북마크)되고, 상단 **⭐ 저장함** 을 누르면 저장한 공고만 모아 봅니다.
  저장한 공고는 나중에 목록에서 사라져도(마감·삭제) 저장함에 그대로 남습니다.
- 미확인 공고가 항상 위로, 그다음 최신순으로 정렬됩니다.
- 지역(부산/전국)·유형(사업화/투자/네트워킹/입주기업/특허·지재권/홍보·마케팅) 필터 제공.
- 읽음 상태는 브라우저에 저장되므로 PC와 폰은 각각 따로 관리됩니다.

## 구조

```
scraper.py   ← 소스별 수집 + 분류 + 범용 자동 크롤러 (requests + BeautifulSoup)
server.py    ← 로컬 앱 서버 (＋사이트 추가 버튼 처리, 앱_시작.bat 이 실행)
custom_sources.json ← 사용자가 추가한 소스 목록
data.json    ← 누적 저장소 (최근 120일)
data.js      ← 대시보드가 읽는 데이터
index.html   ← 대시보드 (단일 파일)
.github/workflows/daily.yml ← GitHub Actions 2시간마다 자동 수집
```

## 소스 추가 (코드 수정 없이)

`custom_sources.json` 파일의 `sources` 목록에 항목을 넣고 저장한 뒤 `업데이트.bat`을 실행하면 됩니다.

**① 바로가기 링크만 추가** (가장 쉬움 — 자동수집은 안 하고 대시보드 하단에 링크 카드로 노출):

```json
{
  "enabled": true,
  "name": "부산창조경제혁신센터",
  "url": "https://ccei.creativekorea.or.kr/busan",
  "region": "부산",
  "mode": "link"
}
```

**② 자동수집 추가** (게시판을 긁어와 공고 카드로 표시). 대상 사이트의 목록 페이지 구조에 맞는 CSS 셀렉터가 필요합니다:

```json
{
  "enabled": true,
  "name": "○○진흥원",
  "region": "부산",
  "mode": "board",
  "list_url": "https://사이트/board/list",
  "base_url": "https://사이트",
  "item_selector": "table tbody tr",
  "title_selector": "td.title a",
  "date_selector": "td.date",
  "link_selector": "td.title a"
}
```

- `item_selector`: 공고 한 줄(행)을 감싸는 요소
- `title_selector` / `link_selector`: 그 안에서 제목/링크(`<a>`)를 가리키는 선택자
- `date_selector`: 등록일 요소 (없으면 행 전체에서 날짜를 자동 추출)
- `base_url`: 링크가 `/board/...`처럼 상대경로일 때 앞에 붙일 주소
- 지역은 `부산` / `전국` / `기타`, 유형은 제목에서 자동 분류됩니다.

> 셀렉터를 직접 찾기 어려우면, 추가하고 싶은 **사이트 주소만 알려주면** 자동수집 설정을 만들어 드립니다.
> `mode`가 `board`인데 수집이 0건이면 셀렉터가 맞지 않는 것이니 `link` 모드로 두거나 문의하세요.
> `enabled`를 `false`로 두면 그 항목은 무시됩니다. 항목을 지우면 해당 소스의 공고도 다음 수집 때 자동 제거됩니다.
